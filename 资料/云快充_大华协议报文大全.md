# 云快充平台协议报文构成文档

## 一、 通信协议结构

### 1.1 应用层报文帧格式

数据帧结构固定，总长度由数据域长度决定，最大不超过 200 字节。

表格

|字段|长度|说明|
|---|---|---|
|**起始标志**|1 Byte|固定为 `0x68`，代表一帧数据的开始|
|**数据长度**|1 Byte|“序列号域+加密标志+帧类型标志+消息体”的字节数之和。不加密时为原数据长度，加密时为加密后数据长度|
|**序列号域**|2 Byte|发送顺序号，从0开始增加；应答包序号与询问包一致。网络断开重连或溢出后归0|
|**加密标志**|1 Byte|仅针对消息体。`0x00`:不加密，`0x01`:3DES加密|
|**帧类型标志**|1 Byte|定义上下行数据帧。充电桩定义为奇数，运营平台定义为偶数|
|**消息体**|N Byte|具体业务数据内容|
|**帧校验域**|2 Byte|从序列号域到数据域的CRC16校验，多项式为 `0x180D`，低字节在前，高字节在后|

### 1.2 数据类型与格式

- **BCD码、BIN码、ASCII**。
- **BIN码**均为低位在前，高位在后。
    
- 小数值均乘倍率上送平台（如电压225.1，保留一位小数，上送值为2251，即 `0x8CB`）。
    
- **CP56Time2a格式**：时间格式标准。

## 三、 各帧类型详细定义

### 6.1 充电桩登录认证 (0x01)

- **功能**：充电桩将桩设置的运营编码上传给运营平台，以实现建立连接关系。
- **传送间隔**：通信中断后上电复位。
- **样例报文**：

text

```
68 (起始标志) 
22 (数据长度) 
0000 (序列号域) 
00 (加密标志) 
01 (类型) 
55031412782305 (桩编码) 
00 (桩类型) 
02 (充电枪数量) 
0F (通信协议版本: V1.5) 
56342E312E353000 (程序版本: v4.1.50) 
01 (网络链接类型) 
010101010101010101 (sim 卡) 
04 (运营商) 
675A (帧校验域)
```

### 6.2 登录认证应答 (0x02)

- **功能**：回复电桩登陆结果。
- **传送间隔**：应答发送。
- **样例报文**：

text

```
68 (起始标志) 
0C (数据长度) 
0000 (序列号域) 
00 (加密标志) 
02 (类型) 
55 03 14 12 78 23 05 (桩编码) 
00 (登陆结果: 成功) 
DA 4C (帧校验域)
```

- **字段定义**：  
    | 序号 | 参数名称 | 数据类型 | 长度 | 备注 |  
    | :---: | :--- | :---: | :---: | :--- |  
    | 1 | 桩编码 | BCD 码 | 7 | 不足 7 位补 0 |  
    | 2 | 登陆结果 | BIN 码 | 1 | 0x00: 登陆成功; 0x01: 登陆失败 |

### 6.3 充电桩心跳包 (0x03)

- **功能**：用于链路状态判断，3次未收到视为网络异常，需重新登陆。
- **传送间隔**：10秒周期上送。

text

```
0001 (序列号域) 
00 (加密标志) 
03 (类型)

```

- **字段定义**：  
    | 序号 | 参数名称 | 数据类型 | 长度 | 备注 |  
    | :---: | :--- | :---: | :---: | :--- |  
    | 1 | 桩编码 | BCD码 | 7 | 不足7位补0 |  
    | 2 | 枪号 | BCD码 | 1 | |  
    | 3 | 枪状态 | BIN码 | 1 | 0x00:正常; 0x01:故障 |

### 4.7、 0xE8 报文整体帧结构

| 字段        | 长度  | 说明                                                                                         |
| --------- | --- | ------------------------------------------------------------------------------------------ |
| **起始标志**  | 1字节 | 固定为 `0x68`                                                                                 |
| **数据长度**  | 1字节 | **数据域字节数** = 序列号域(2) + 加密标志(1) + 帧类型标志(1) + 消息体长度。  <br>总长度计算公式为：`201 + n * 22` （n为充电时段数）。 |
| **序列号域**  | 2字节 | 发送顺序号，从0开始顺序增加。低字节在前，高字节在后。                                                                |
| **加密标志**  | 1字节 | Bit7：0不加密，1加密(SM4)。  <br>**长度扩展说明**：若数据长度 > 256 字节，需借用该字节的 Bit0 表示数据长度的高8位。                |
| **帧类型标志** | 1字节 | `0xE8`                                                                                     |
| **消息体**   | N字节 | 包含基础交易信息及时段明细，详见下方字段表。                                                                     |
| **帧校验域**  | 2字节 | CRC16校验，校验多项式 `0x180D`，低字节在前，高字节在后。                                                        |

### 三、 关键注意事项

1. **动态长度与跨256字节处理**：  
    平台解析时需根据第42字段“充电时段数n”动态计算长度。当 `n > 2` 时（即总长 `201 + 3*22 = 267` 超过256字节），**数据长度域**仅能存放低8位，需将超出的高8位存放在**加密标志位**的 Bit0 中。
    
2. **替代关系**：  
    设备若支持并启用 `0xE8` 增强版交易记录，**必须去掉原云快充标准协议中的 `0x3B` 交易记录上报**，统一使用 `0xE8`。
    
3. **上报流程**：  
    `0xE8` 在充电结束后由充电桩主动发送至平台，平台接收并进行结算扣费处理后，回复 `0xE9` 交易记录回复。充电桩收到 `0xE9` 后方可清除本地该订单缓存。

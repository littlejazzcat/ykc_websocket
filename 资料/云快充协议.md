# 云快充充电桩与平台交互协议总结文档

## 一、协议概述

本协议规定了充电桩与云快充平台之间数据交互的流程、格式和内容。协议整体依据国网 104 充电桩规约，新增数据部分参照 GBT-27930。协议适用于交直流充电桩，交流桩部分无需上送的数据项在协议中均有标注。

## 二、通信接口与连接

- **通信方式**：基于 TCP/IP Socket 的长连接通信。
- **网络支持**：支持有线网络、无线 GPRS 或通过集中器连接。
- **服务器配置**：支持直接 IP 和域名解析。测试服务器地址为 121.199.192.223，端口为 8768。
- **接入流程**：桩企开发 -> 平台分配测试设备编号及账户 -> 测试平台联调数据帧 -> APP及刷卡流程调试 -> 正式环境配置。

## 三、应用层报文帧格式

数据帧结构固定，总长度由数据域长度决定，最大不超过 200 字节。

|起始标志|数据长度|序列号域|加密标志|帧类型标志|消息体|帧校验域|
|---|---|---|---|---|---|---|
|1字节|1字节|2字节|1字节|1字节|N字节|2字节|

- **起始标志**：固定为 0x68。
- **数据长度**："序列号域+加密标志+帧类型标志+消息体"的字节数之和。
- **序列号域**：发送顺序号，从0增加，应答包序号一致，断线重连或溢出后归0。
- **加密标志**：仅针对消息体。0x00不加密，0x01 3DES加密。
- **帧校验域**：CRC16校验，多项式 0x180D，低字节在前高字节在后。
- **数据格式**：包含 BCD码、BIN码（低位在前高位在后）、ASCII。小数值乘倍率上送。

## 六、核心名词解释

- **充电卡**：默认M1卡，物理卡号用于交互，逻辑卡号用于屏幕显示。
- **交易流水号**：一次充电唯一标识。格式：桩号(7bytes)+枪号(1byte)+年月日时分秒(6bytes)+自增序号(2bytes)。
- **计损比例**：非零时，充电度数需加上此比例上送计损电度。（目前平台不支持，置0）。
- **桩/枪编号**：桩编号由平台生成7字节，枪号为桩编号+1字节枪号（如 /01, /02）。

## 七、协议需知（核心注意事项）

1. **启动回复**：收到平台启充指令后，电桩需在90秒内回复成功并上送"充电中"状态实时数据，否则平台关闭订单。
2. **停充指令**：充电中平台下发停充，桩必须停止充电，否则后续电量无法入账。
3. **状态一致性**：充电中不得上送"空闲"状态（超过两帧会被判异常订单）；拔枪需立即上送交易记录结束订单。
4. **流水号一致**：鉴权回复、实时数据、交易记录中的流水号必须一致。
5. **交易记录上送**：充电结束后30秒内必须上送交易记录，分时电量必须与平台计费模型一致。

## 八、附录

- **停止原因代码表**：涵盖 0x40~0x79 等，分为"充电完成"、"充电启动失败"（如余额不足、BMS超时等）、"充电异常中止"（如断路器跳位、温度异常等）。
- **CRC16校验**：采用查表法计算，提供高字节与低字节常量数组表。


# 云快充协议 0x3B 交易记录帧完整构成

## 一、帧头信息

|项目|内容|
|---|---|
|**帧类型码**|`0x3B`|
|**传送方向**|充电桩 → 运营平台|
|**传送间隔**|主动上送（充电结束后发送）|
|**功能**|充电桩在网络正常情况下，主动发送结算账单，直到运营平台响应账单上传成功。每次接收到启机命令并已执行启机过程，**无论启机成功与否**，都需在订单结束充电后生成账单上传|

## 五、补充说明

1. **应答帧**：运营平台收到 0x3B 后需回复 **0x40（交易记录确认）**，充电桩收到确认后本地删除该账单
    
2. **BIN码规则**：低位在前，高位在后
3. **小数处理**：协议中小数值均乘倍率上送（如电压225.1保留一位小数，上送值为2251）
    
4. **大华补充协议替代**：若设备支持 **0xE8 增强版交易记录**，则需去掉 0x3B 上报，改用 0xE8（支持分时段明细，报文长度可变：201 + n×22）

### 六、校验规则
### 1. 校验规则说明

- **校验范围**：从“序列号域”开始，到“数据域（消息体）”结束的所有字节。
- **校验多项式**：`0x180D`
    
- **字节序**：计算结果为两字节，**低字节在前，高字节在后**。
- **计算方式**：采用**查表法**进行计算。

### 2. 计算函数代码（C语言示例）

c

```
WORD ModbusCRC(BYTE *pData, BYTE len)
{
    BYTE byCRCHi = 0xff;
    BYTE byCRCLo = 0xff;
    BYTE byIdx;
    WORD crc;
    
    while (len--)
    {
        byIdx = byCRCHi ^ *pData++;
        byCRCHi = byCRCLo ^ gabyCRCHi[byIdx];
        byCRCLo = gabyCRCLo[byIdx];
    }
    
    crc = byCRCHi;
    crc << 8;
    crc += byCRCLo;
    
    return crc;
}
```

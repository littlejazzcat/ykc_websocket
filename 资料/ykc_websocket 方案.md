# ykc_websocket 中转工具方案

## 一、项目定位

一个 PC 端运行的充电桩通信报文监控与透传工具。部署在充电桩与云快充平台之间，通过**串联模式**监听并转发双方的 TCP 通信，同时将解析后的报文通过 WebSocket 实时推送到浏览器前端做可视化展示。

**核心场景：** 开发调试、协议分析、故障排查。

## 二、双向数据流架构

```
                              ┌────────────── 中转工具 ──────────────┐
                              │                                     │
  充电桩                       │  ┌──────────────────────────────┐    │       真实云快充平台
  (接入端)                     │  │   TCP Server (asyncio)        │    │    (121.199.x.x:8768)
     │                         │  │                              │    │          │
     │  ◄── TCP 全双工 ────►  │  │   桩连接 ◄─────► 平台连接    │    │          │
     │                         │  │   (Session 一对一绑定)       │    │          │
     │   ① 上行帧 (type:奇数)   │  │                              │    │          │
     │ ──────────────────────► │  │  解析帧头(0x68)              │   │  ② 透传上行 │
     │                         │  │  → 识别帧类型                 │   │ ────────► │
     │                         │  │  → 序列化 → WebSocket        │   │          │
     │                         │  │  → 透传至平台                 │   │          │
     │                         │  │                              │   │          │
     │   ④ 透传下行             │  │                              │   │  ③ 平台响应 │
     │ ◄────────────────────── │  │  解析帧头(0x68)              │   │ ◄──────── │
     │                         │  │  → 识别帧类型                 │   │          │
     │                         │  │  → 序列化 → WebSocket        │   │          │
     │                         │  │  → 透传回充电桩               │   │          │
     │                         │  └──────────────┬───────────────┘   │          │
     │                         │                 │                   │          │
     │                         │          FastAPI Server             │          │
     │                         │         ┌────WebSocket────┐         │          │
     │                         └─────────┤  双向报文推送    ├─────────┘          │
     │                                   │  带方向标记      │                    │
     │                                   │  ↑上行 / ↓下行   │                    │
     │                                   └────────┬─────────┘                    │
     │                                            │                              │
     │                                      ⑤ 实时数据流                          │
     │                                            │                              │
     │                                     ┌──────▼──────┐                       │
     │                                     │   浏览器     │                       │
     │                                     │  实时表格    │                       │
     │                                     │  (WebSocket) │                       │
     │                                     └─────────────┘                       │
```

**方向定义：**
| 方向 | 说明 | 帧类型码特征 |
|------|------|-------------|
| ↑ 上行 | 充电桩 → 平台 | 奇数 (充电桩定义) |
| ↓ 下行 | 平台 → 充电桩 | 偶数 (平台定义) |

**透传规则：** 中转工具对上/下行报文均原样透传，不做任何修改。平台连接在充电桩首次连入时建立，两者生命周期绑定。

## 三、TCP 帧解析设计

### 3.1 帧结构（云快充/大华通用）

| 字段 | 起始标志 | 数据长度 | 序列号域 | 加密标志 | 帧类型标志 | 消息体 | 帧校验域 |
|------|---------|---------|---------|---------|-----------|--------|---------|
| 字节 | 1 | 1 | 2 | 1 | 1 | N | 2 |

- 起始标志：固定 `0x68`
- 数据长度：序列号域 + 加密标志 + 帧类型标志 + 消息体 的字节数
- 帧校验域：CRC16，多项式 `0x180D`，低字节在前

### 3.2 解析状态机

TCP 是流式协议，必须同时处理**粘包**（多帧连在一起）和**半包**（一帧被截断为多段到达）：

```
                    ┌──────────────┐
       收到数据 ───►│   WAIT_HEAD  │
                    │  等待 0x68   │
                    └──────┬───────┘
                           │ 找到 0x68
                           ▼
                    ┌──────────────┐
                    │  READ_LENGTH │
                    │  读取长度字段  │
                    └──────┬───────┘
                           │ 得到帧总长 = 1 + 数据长度 + 2(CRC)
                           ▼
                    ┌──────────────┐
                    │  WAIT_BODY   │
                    │  等待剩余字节  │
                    └──────┬───────┘
                           │ 收齐完整帧
                           ▼
                    ┌──────────────┐
                    │  VERIFY_CRC  │
                    │  CRC16 校验   │
                    └──────┬───────┘
                      ╱         ╲
                   通过          失败
                    │            │
                    ▼            ▼
               ┌─────────┐  ┌──────────┐
               │ 分发帧   │  │ 丢弃 +   │
               │ (正常)   │  │ 错误日志  │
               └────┬─────┘  └──────────┘
                    │            │
                    └─────┬──────┘
                          ▼
                    返回 WAIT_HEAD
                   (处理缓冲区剩余字节)
```

**关键处理逻辑：**

1. **半包**：当前缓冲区数据不足一帧长，继续等待下次 `read()`，不丢数据。
2. **粘包**：解析完一帧后，缓冲区还有剩余字节，循环回到 WAIT_HEAD 继续解析。
3. **异常帧**：CRC 校验失败或起始标志不在预期位置，记录日志并丢弃该帧，尝试从下一个 `0x68` 重新同步。

### 3.3 大华动态长度帧（0xE8 交易记录增强版）

大华 `0xE8` 帧的数据长度由"充电时段数 n"决定：`总长度 = 201 + n × 22`。

解析流程：
1. 先按固定头（前 42 个字段）解析到"充电时段数"字段
2. 读取 n 值
3. 按 `201 + n * 22` 验证实际帧长度是否匹配
4. 若 n 值异常（如超过合理范围），记录警告并丢弃

## 四、连接管理与会话模型

### 4.1 Session（会话）

每个充电桩连接对应一个 Session，内部维护：

```
Session
├── 桩编号         # 从登录帧(0x01)消息体中提取（7字节 BCD）
├── 桩连接
│   ├── reader    # asyncio.StreamReader
│   └── writer    # asyncio.StreamWriter
├── 平台连接
│   ├── reader    # asyncio.StreamReader
│   └── writer    # asyncio.StreamWriter
├── 在线状态       # 登录中 / 在线 / 离线
└── ws_队列        # asyncio.Queue → FastAPI WebSocket
```

### 4.2 生命周期

```
充电桩 TCP 连接进入
    │
    ▼
创建 Session，建立平台连接
    │
    ├── 桩连接断开 → 关闭平台连接，销毁 Session
    │
    ├── 平台连接断开 → 尝试重连平台 (指数退避, 最多3次)
    │                   └── 仍失败 → 关闭桩连接，销毁 Session
    │
    └── 正常通信中 → 双向透传 + 报文解析 + WebSocket 推送
```

### 4.3 平台连接建立时机

**充电桩成功登录后建立。** 即收到 `0x01` 登录认证帧时，从中提取桩编号，然后建立到真实平台的 TCP 连接，转发登录帧。这样确保 Session 的桩编号在连接建立时就已知。

## 五、技术选型

| 层 | 技术 | 说明 |
|----|------|------|
| TCP 通信 | Python `asyncio` | `start_server` 监听充电桩连接，`open_connection` 连接平台 |
| HTTP / WebSocket | FastAPI + `websockets` | HTTP 渲染前端页面；WebSocket 推送报文 |
| 前端 | 原生 HTML/CSS/JS | 无框架依赖，单文件完成 |
| 协议解析 | 纯 Python | 按帧结构逐字节解析，CRC16 查表法 |

**依赖清单（requirements.txt）：**
```
fastapi
uvicorn[standard]
websockets
```

```
│  ykc_websocket 报文监控                    [连接状态●]  │

│  桩编号: 1234567 | 平台: 121.199.192.223:8768           │
├─────────────────────────────────────────────────────────┤

├──────┬────┬────────┬────────────────────────────────────┤
│ 时间  │方向 │ 帧类型  │ 关键字段                           │  ← 表头，可排序
├──────┼────┼────────┼────────────────────────────────────┤

│14:32:01│ ↓  │ 0x02   │ 登录应答: 允许接入                 │  ← ↓橙色
│14:32:05│ ↑  │ 0x05   │ 计费模型验证请求                   │
│14:32:05│ ↓  │ 0x06   │ 计费模型验证应答: 一致             │
│14:32:10│ ↑  │ 0x03   │ 心跳包                            │

│  ...   │ .. │  ...   │ ...                               │

```

### 6.2 核心功能

- **实时报文表格**：新报文自动追加到表格顶部/底部，不同方向用不同底色区分
- **方向过滤**：点击按钮切换显示全部/仅上行/仅下行
- **帧类型过滤**：按帧类型码快速筛选（心跳、实时数据、交易记录等）
- **报文详情**：点击某一行展开，查看完整字段明细（hex 值 + 解析值）
- **自动滚动**：默认跟随最新报文滚动，手动滚动时暂停自动跟随
- **导出功能**：将当前显示的报文导出为 CSV 或 JSON

## 七、实现优先级

| 阶段 | 内容 | 说明 |
|------|------|------|
| **P0 核心** | TCP Server + 帧解析状态机 | 处理粘包/半包，完整解析一帧 |
| **P0 核心** | 平台连接 + 上行透传 | 桩 → 中转 → 平台 |
| **P0 核心** | WebSocket 推送 + 前端基础表格 | 能实时看到报文 |
| **P1 重要** | 平台响应读循环 + 下行透传 | 平台 → 中转 → 桩 + 前端 |
| **P1 重要** | 前端方向/类型过滤 | 快速定位目标报文 |
| **P2 增强** | 大华协议帧解析 (0xC0/C2/D0/E8) | 动态长度帧特殊处理 |
| **P2 增强** | 报文详情展开 + hex 展示 | 调试时逐字节查看 |
| **P3 完善** | 日志回放、导出、错误统计 | 生产辅助功能 |

## 八、注意事项

1. **加密帧处理**：加密标志为 `0x01` 时消息体经过 3DES 加密，中转工具无法解析明文字段，前端展示 raw hex 并在字段区标注"已加密"。
2. **CRC16 校验**：只用于帧完整性验证，校验失败的帧标记为异常但不影响透传（平台端会自行校验处理）。
3. **端口配置**：充电桩连入的监听端口可配置；平台目标地址和端口可配置。
4. **多桩并发**：当前阶段按单桩设计，架构预留 Session 管理机制，多桩接入时只需扩展 Session 列表和前端桩编号选择器。
5. **离线缓存**：不缓存报文。若前端 WebSocket 断开重连，之前的历史报文丢失。需要历史数据的场景留待 P3 日志回放功能。
